Description

This example sets two Structures and makes copies of these Structures using the setq and the copy function. It then uses the isIdentical function to compare the copies of the two Structure.

 

Details

(setq A #{X:1 Y: 2 Z:3}) Returns: #<Structure 123456>
(setq B #{X:1 Y: 2 Z:3}) Returns: #<Structure 234567>

C has a different Structure Id although the contents of C and A are the same.

(setq C (copy A )) Returns: #<Structure 345678>

D and A have the same Structure Id since the setq function assigns a pointer to the same memory location of the source object.

(setq D A) Returns: #<Structure 123456>

A and B have different Structure Ids.

(isIdentical A B) Returns: false
(isIdentical B C) Returns: false

D and A access the same memory location.

(isIdentical D A) Returns: true

Notes and Hints

When you use the setq function on a collection data type such as a Structure, a pointer to the data location (the object id) is copied to the target container (target). Therefore, the source and target container are pointing to the same data. The isIdentical function returns true for the source and target containers in the setq function.

However if the copy function was used, it produces a copy of the Structure (target) in a different memory location. Therefore, the isIdentical function returns false for the source and target containers in the copy function.
